# drunk-pulumi-azure-components

Reusable Pulumi `ComponentResource` library targeting Azure (`@pulumi/azure-native` 3.x) and Azure AD. Published from `bin/` after build.

## Project Layout

| Path | Purpose |
|------|---------|
| `src/base/` | Shared base classes â€” `ResourceBuilder`, `AppResources`, `types.ts`. Every component flows through these. |
| `src/{aks,apim,app,azAd,database,logs,services,storage,vault,vm,vnet}/` | Domain components grouped by Azure service. |
| `src/common/`, `src/helpers/` | Cross-cutting utilities (naming, tags, RBAC role IDs, network/IP math via `netmask`, PGP via `openpgp`). |
| `__tests__/` | Jest unit tests. Mirrors `src/` layout. |
| `pulumi-test/` | Integration fixture â€” exercises components against a real Pulumi stack. |

| `.tasks/` | Build/release scripts (`npm-package.ts`, `update-tsconfig.ts`, `generate-docs.ts`). |

## Conventions

- **Component shape**: every public component extends `pulumi.ComponentResource`, accepts a typed `Args` interface, registers child resources with `{ parent: this }`, and calls `this.registerOutputs(...)` at the end of the constructor. New components must follow this pattern â€” read a sibling under the same `src/<domain>/` folder before writing one.
- **Inputs / outputs**: use `pulumi.Input<T>` for constructor args and `pulumi.Output<T>` for exposed properties. Never `await` outputs â€” use `.apply()`.
- **Naming + tags**: pull from `src/common/` helpers; do not hard-code resource names or tag objects.
- **RBAC**: role definition IDs come from the helper map in `src/common/`/`src/azAd/`. Do not paste raw GUIDs into components.
- **Formatting**: Prettier config lives in `package.json` (`singleQuote`, `printWidth: 120`, `trailingComma: all`, `semi: true`). ESLint via `eslint.config.js`.
- **Tests**: Jest + `ts-jest`. Co-locate by mirroring `src/` path inside `__tests__/`. Test the public component contract (inputs â†’ registered child resources + outputs), not internal helpers.
- **Imports**: TypeScript `6.0.3`, `tsconfig.json` is regenerated by `.tasks/update-tsconfig.ts` â€” do not edit `tsconfig.json` by hand for new component dirs; add the dir and re-run `pnpm run update-tsconfig`.

## Editing Rules

- **Before changing a component's public surface** (constructor args, registered outputs, exported types), run impact analysis (see GitNexus block below). Consumers in `pulumi-test/` and downstream stacks break silently otherwise.
- **Security-sensitive surface**: `src/vault/`, `src/azAd/`, anything touching `keyvault-*`, RBAC role assignments, network ACLs (`netmask`), or `openpgp`. Treat changes here as security-reviewable â€” no widening of IAM scope, no public endpoints, no plaintext secret handling without explicit ask.
- **Build output**: never edit `bin/`. If `bin/` looks wrong, fix the source and rebuild.
- **Lockfile**: mutate `pnpm-lock.yaml` only via `pnpm install` / `pnpm update`. Never hand-edit.
- **Azure SDK shapes shift between minor versions of `@pulumi/azure-native`**. When in doubt about a resource's property names or enum values, consult Microsoft Learn / Pulumi registry rather than guessing.
